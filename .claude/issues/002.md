# 概要
issue 001 で 2025-07 にレート制限対応（リトライ強化・バッチ処理・待機時間調整）を実施したが、
2026-06 時点で再検証したところ、**1銘柄の単発 `history()` 呼び出しでも `YFRateLimitError` が即発生** する状態。
これは呼び出し回数の問題ではなく、yfinance ライブラリと Yahoo 側の Cookie/crumb 認証ハンドシェイクの噛み合わせ問題と推測される。

# 環境
- Python 3.12.6 (venv)
- yfinance 0.2.54
- requests 2.31.0
- tenacity 9.0.0
- pandas 2.2.1

# 再現コード
```python
import yfinance as yf
t = yf.Ticker('2914.T')
df = t.history(period='5d')
```
→ `yfinance.exceptions.YFRateLimitError: Too Many Requests. Rate limited. Try after a while.`

# 観察
- 単発呼び出しで即エラーのため、issue 001 で実装したバッチ間待機/リトライ間隔調整では回避不可能なレイヤの問題
- yfinance 0.2.5x 系で広く報告されている既知問題（Cookie/crumb 処理の不整合）

# 検証方針
1. `yfinance` を最新版へアップグレードして再現性確認
2. 改善しない場合は `curl_cffi` セッションを使ったブラウザ偽装に切替
3. それでも駄目なら別データソース検討

# 付随する技術的負債（本 issue のスコープ外）
- `requirements.txt` が大幅に古い（yfinance==0.1.63 表記、tenacity/python-dotenv/pandas 未記載）
- 別 issue で更新が必要
