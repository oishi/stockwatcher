# 概要
yfinance APIのレート制限により、108銘柄の株価データ取得時に「Too Many Requests」エラーが発生する問題を解決しました。

# 問題の詳細
- Yahoo Finance側が2024-2025年にレート制限を大幅強化
- 従来の0.5秒待機では不十分
- 108銘柄の連続取得でIPベースのブロックが発生
- リトライ機能も制限に対応できていない

# 実装した解決策
## 1. リトライ機能の強化
- リトライ回数: 3回 → 5回
- 待機時間: 4-10秒 → 10-60秒（指数バックオフ）

## 2. 待機時間の改善
- 固定0.5秒 → ランダム5-10秒
- レート制限エラー検出時: 30-60秒の追加待機

## 3. バッチ処理システムの追加
- デフォルトバッチサイズ: 5銘柄
- バッチ間待機: 30-60秒のランダム待機
- `--batch-size`オプションで調整可能

## 4. エラーハンドリングの強化
- レート制限エラーの特別扱い
- より詳細なログ出力

# 変更したファイル
- `src/python/price_updater.py`: メインロジックの改善
- `bin/update.sh`: バッチサイズオプションの追加
