## 詳細
yfinance や GAS など外部依存が多いスクリプトであり、まずは外部 I/O をモックして
純粋なロジック（データ整形・引数/期間パース・GAS 送信ペイロード生成・エラーハンドリング分岐）を
対象にした単体テストから始めるのが現実的。ネットワークに依存しない高速なテストを優先する。

### テスト対象の候補（実装確認のうえ確定）
- 引数パース（銘柄指定／`--period`／`--batch-size` 等）
- yfinance から取得した DataFrame → GAS 送信用 JSON ペイロードへの変換
- レート制限・通信エラー時のリトライ／分岐ロジック（外部呼び出しはモック）
- issue 004 で追加する GAS 疎通チェックの判定ロジック

### 方針判断
- テストフレームワークは **pytest** を推奨（デファクト・モック容易・記述が簡潔）
- ネットワークアクセスを伴う結合テストはスコープ外とし、`unittest.mock` / `monkeypatch` で外部依存を遮断
- `requirements.txt` とは分離し、`requirements-dev.txt`（または extras）に pytest 系を記載


## 方針
1. `tests/` ディレクトリを新設し pytest を導入する
2. 外部依存（yfinance / requests）をモックする方針を固める
3. まず純粋ロジックの単体テストを 1〜数本書き、リポジトリ直下からの実行手順を確立する
4. CLAUDE.md のテスト方針記述を実態に合わせて更新する
