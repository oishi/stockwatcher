# 概要
`requirements.txt` の記載が実際の venv と大きく乖離しており、別マシンで `git clone` 後に
`pip install -r requirements.txt` しても `price_updater.py` が動かない状態。
issue 002 でレート制限解消のため yfinance を 1.4.1 にアップグレードしたことも反映する。

# 問題点
- `yfinance==0.1.63` 記載（実体は 1.4.1。メジャー差・レート制限問題のため使えない）
- `requests==2.26.0` 記載（実体 2.31.0）
- スクリプトが import している `python-dotenv` `pandas` `tenacity` が **未記載**

# 対応方針
`price_updater.py` が直接 import する依存を実バージョンでピン留め:
```
yfinance==1.4.1
requests==2.31.0
pandas==2.2.1
python-dotenv==1.0.1
tenacity==9.0.0
```
※ `curl_cffi` は yfinance の依存として自動導入されるため明記しない。

# 検証（TDD相当）
クリーンな一時 venv を作成し `pip install -r requirements.txt` が成功すること、
および `python -c "import yfinance, requests, pandas, dotenv, tenacity"` が通ることを確認。
